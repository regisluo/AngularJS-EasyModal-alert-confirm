Angular-EasyModal-alert-confirm

This easyModal can be used in AngularJS-base(with Bootstrap) web application, which is an alternative of JS alert and confirm dialog with enhanced functionality(html template,css,callback,external controller), more flexible, better user experience.

Dependencies

Requires AngularJS v1.4.0+ and Bootstrap's CSS.

Alert Example

emodal.alert("<strong>Title</strong> <p></p>Disappear in 2 seconds",2000);

Confirm Example

emodal.confirm("Are you sure to delete?<p></p>It is not recoverable!",
               function(){
                   //do something if OK is pressed
                   $scope.data='You have pressed OK!';
               },
               function(){
                   //do something if Cancel is pressed
                   $scope.data='You have pressed Cancel!';
               }
           );

Modal Example

var okCallbackFun = function (arg) {
            $scope.data = '[ok callback] Parameter: '+arg.id;
        };
var cancelCallbackFun = function (arg) {
            $scope.data = '[cancel callback] Parameter: '+arg.info;
        };
$scope.showModal3 = function(){
            emodal.modal({template:'Messages go here!',title:'Callback Function',
                okCallback:okCallbackFun,okArgs:{id:1},
                cancelCallback:cancelCallbackFun,cancelArgs:{info:'i.e. Help text'}
            });
        };

Usage

1. alert

This aims to provide the function like native JS alert, but with more customized properties.
There are maximum 3 arguments with four calling types:
emodal.alert(string);
emodal.alert(string, string);
emodal.alert(string, number);
emodal.alert(string, string, number);

1.1 one argument

emodal.alert("Messages go here!");

1.2 two arguments

//string+string emodal.alert("Messages go here!","New Button Label");
OR //string+number(millionseconds)
emodal.alert("Title

Disappear in 2 seconds",2000);

1.3 three arguments
//message + new label name + millionseconds that the modal vanish emodal.alert("Title

Disappear in 2 seconds","New Button Label",2000);

2. confirm

This aims to provide the function like traditional JS confirm, but with more customized properties.
There are totally 5 arguments with generally two calling types:
emodal.confirm(string /required/, function /required/, function /optional/);
OR
emodal.confirm(string /required/, function /required/, function /optional/, object /optional/, object /optional/);

2.1 three (2 required + 1 optional) arguments

string: message shown in the modal
function: callback function if click OK
function:(optional) callback function if click Cancel 

emodal.confirm("Are you sure to delete?<p></p>It is not recoverable!",
    function(){
        // if OK is clicked
    },

    function(){
         // if Cancel is clicked
     }

);

2.2 five (2 required + 3 optional) arguments

string: message shown in the modal
function: callback function if press OK
function:(optional) callback function if press Cancel
object:(optional) "this" object inside okCallback function
object:(optional) "this" object inside cancelCallback function

3. modal

Modal aims to provide more functionality not as simple alert and confirm dialog.
Generally the argument passed in is an object with the following arguments:

example:

emodal.modal("../partials/externalTemplate.html",{title:'External Template'});
